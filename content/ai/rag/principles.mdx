# RAG 检索增强生成原理

> RAG 让大模型拥有了"查资料"的能力，是解决幻觉问题的有效方案。

## 1. RAG 工作流程

```
用户提问
  → 问题向量化
  → 向量检索相关文档
  → 文档 + 问题 → LLM
  → 生成回答
```

## 2. 核心组件

### 嵌入模型

```python
from openai import OpenAI
client = OpenAI()

response = client.embeddings.create(
    model="text-embedding-3-small",
    input="什么是RAG？"
)
vector = response.data[0].embedding  # 1536维向量
```

### 向量数据库

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| Pinecone | 全托管 | 快速起步 |
| Weaviate | 开源 | 私有部署 |
| Chroma | 轻量 | 本地开发 |

## 3. 个人分析

- RAG 的关键不是模型，而是文档质量与检索精度
- Chunk 策略（分块方式）对效果影响极大
- Hybrid Search（向量+关键词）比纯向量检索效果更好
