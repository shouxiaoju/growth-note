# React 状态管理方案对比

> 选对状态管理方案，能让项目架构清晰、维护轻松。

## 1. 方案概览

| 方案 | 类型 | 适用场景 | 学习曲线 |
|------|------|----------|----------|
| useState/useReducer | 内置 | 组件级状态 | 低 |
| Context + useReducer | 内置 | 小型应用 | 中 |
| Zustand | 外部 | 中型应用 | 低 |
| Jotai | 外部 | 原子化状态 | 低 |
| Redux Toolkit | 外部 | 大型应用 | 高 |

## 2. Zustand 示例

```javascript
import { create } from 'zustand';

const useStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  reset: () => set({ count: 0 }),
}));

function Counter() {
  const { count, increment, reset } = useStore();
  return (
    <div>
      <span>{count}</span>
      <button onClick={increment}>+1</button>
      <button onClick={reset}>重置</button>
    </div>
  );
}
```

## 3. 个人分析

- 2026 年前端状态管理趋势：轻量化、原子化
- 中小项目推荐 Zustand，大项目用 Redux Toolkit
- 80% 的场景其实只需要 React 内置的 useState + Context
