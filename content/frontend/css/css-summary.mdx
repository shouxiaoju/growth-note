## CSS 基础与选择器

### CSS 选择器有哪些？优先级怎么算？

**选择器分类**

| 类别 | 选择器 | 示例 |
|------|--------|------|
| 简单选择器 | 标签、类、ID、通配 | `div` `.box` `#app` `*` |
| 属性选择器 | 属性匹配 | `[type="text"]` `[href^="https"]` |
| 伪类选择器 | 状态/位置 | `:hover` `:first-child` `:nth-child(n)` `:not()` |
| 伪元素选择器 | 虚拟元素 | `::before` `::after` `::first-line` `::selection` |
| 组合选择器 | 关系组合 | `A B`(后代) `A>B`(子代) `A+B`(相邻兄弟) `A~B`(通用兄弟) |

**优先级计算（从高到低）**

| 等级 | 选择器 | 权重值 |
|------|--------|--------|
| `!important` | — | 最高，覆盖一切 |
| 内联样式 | `style="..."` | (1,0,0,0) |
| ID 选择器 | `#app` | (0,1,0,0) |
| 类/伪类/属性 | `.box` `:hover` `[type]` | (0,0,1,0) |
| 标签/伪元素 | `div` `::before` | (0,0,0,1) |
| 通配符 | `*` | (0,0,0,0) |

```css
/* 优先级比较示例 */
#nav .item a        /* (0,1,1,1) */
.nav .item .link    /* (0,0,3,0) */
div p::first-line   /* (0,0,0,3) */

/*  同权重时后者覆盖前者 */
.text { color: red; }
.text { color: blue; }  /* 最终蓝色 */

/*  !important 慎用，破坏优先级链 */
.text { color: red !important; }  /* 强制覆盖，但维护噩梦 */
```

>  **面试口诀**：`!important` > 内联 > ID > 类/伪类/属性 > 标签/伪元素 > 通配

### CSS 继承有哪些？哪些属性不继承？

```css
/* 可继承的属性（与文字/列表相关） */
color, font-family, font-size, font-weight, font-style,
line-height, text-align, text-indent, text-transform,
letter-spacing, word-spacing, white-space,
list-style, list-style-type, visibility, cursor

/* 不可继承的属性（与盒模型/布局相关） */
width, height, margin, padding, border,
display, position, float, clear,
overflow, min-width, max-width,
background, z-index, opacity, box-shadow
```

> **常见坑**：`color` 可继承但 `a` 标签不继承父元素 `color`，浏览器默认给 `a` 设了蓝色。

```css
/*  让 a 标签继承父元素颜色 */
a { color: inherit; }
```

## 盒模型

### 标准盒模型 vs 怪异盒模型

```
标准盒模型（content-box）：
┌─────────────────────────────┐
│           margin             │
│  ┌────────────────────────┐ │
│  │        border           │ │
│  │  ┌──────────────────┐  │ │
│  │  │     padding      │  │ │
│  │  │  ┌────────────┐  │  │ │
│  │  │  │  content   │  │  │ │
│  │  │  │  width=100 │  │  │ │
│  │  │  └────────────┘  │  │ │
│  │  └──────────────────┘  │ │
│  └────────────────────────┘ │
└─────────────────────────────┘
总宽度 = margin + border + padding + width
设置 width=100px → 内容区100px，盒子更宽

怪异盒模型（border-box）：
┌─────────────────────────────┐
│           margin             │
│  ┌────────────────────────┐ │
│  │  width=100（含border+padding） │
│  │  ┌──────────────────┐  │ │
│  │  │    padding       │  │ │
│  │  │  ┌────────────┐  │  │ │
│  │  │  │  content   │  │  │ │
│  │  │  └────────────┘  │  │ │
│  │  └──────────────────┘  │ │
│  └────────────────────────┘ │
└─────────────────────────────┘
总宽度 = margin + width
设置 width=100px → 整个盒子100px，内容区自动缩小
```

```css
/* 全局推荐设置 */
*, *::before, *::after {
  box-sizing: border-box;
}

/* 切换盒模型 */
.box-standard { box-sizing: content-box; }  /* 默认 */
.box-border   { box-sizing: border-box; }   /* 推荐 */
```

> **面试追问**：为什么推荐 `border-box`？
> 设置宽度就是最终宽度，加 padding/border 不会撑大盒子，布局更可控。

### margin 塌陷与合并怎么解决？

**margin 合并（垂直方向）**

```
两个相邻块级元素的上下 margin 会合并，取较大值

┌──────┐     ┌──────┐
│  A   │     │  A   │
│margin-bottom:20px│
└──────┘     └──────┘
   ↕ 20px（不是40px）    ↕ 30px（取较大值）
┌──────┐     ┌──────┐
│  B   │     │  B   │
│margin-top:20px│    │margin-top:30px│
└──────┘     └──────┘
```

**margin 塌陷（父子元素）**

```
父元素没有 border/padding/overflow 时
子元素的 margin-top 会"穿透"到父元素外部

┌──────────────┐
│  父元素       │  ← 子元素的 margin-top 跑到这里了
│  ┌────────┐  │
│  │ 子元素  │  │
│  └────────┘  │
└──────────────┘
```

```css
/* 解决 margin 塌陷的4种方式 */

/* 方案1：父元素加 border */
.parent { border-top: 1px solid transparent; }

/* 方案2：父元素加 padding */
.parent { padding-top: 1px; }

/* 方案3：父元素 overflow: hidden（BFC） */
.parent { overflow: hidden; }

/* 方案4：父元素加 ::before 伪元素 */
.parent::before {
  content: '';
  display: table;
}
```

> **注意**：margin 合并只发生在**垂直方向**的**块级元素**之间，水平方向不合并，行内元素不合并。

## BFC（块级格式化上下文）

### 什么是 BFC？怎么触发？有什么用？

**BFC 是一个独立的渲染区域，内部元素的布局不影响外部**

**触发条件（满足任一即可）**

| 触发方式 | CSS 属性 |
|---------|---------|
| 根元素 | `<html>` 本身就是 BFC |
| 浮动 | `float: left/right`（非 none） |
| 绝对定位 | `position: absolute/fixed` |
| 行内块 | `display: inline-block` |
| 弹性/网格 | `display: flex/inline-flex/grid/inline-grid` |
| 表格相关 | `display: table/table-cell/table-caption` |
| overflow | `overflow: hidden/auto/scroll`（非 visible） |
| 多列 | `column-count` 或 `column-width` |
| contain | `contain: layout/content/paint` |

**BFC 的三大作用**

```css
/* ① 解决 margin 塌陷 */
.parent { overflow: hidden; }  /* 触发 BFC，子 margin 不穿透 */

/* ② 清除浮动（父元素高度塌陷） */
.container { overflow: hidden; }  /* 触发 BFC，包含浮动子元素 */

/* ③ 阻止元素被浮动覆盖（自适应布局） */
.sidebar { float: left; width: 200px; }
.main { overflow: hidden; }  /* 触发 BFC，不与浮动重叠，自适应剩余宽度 */
```

> **面试答题**：先说定义 → 再说触发方式（3-4个即可）→ 最后说作用（3个场景）

## 浮动与清除

### 清除浮动有哪些方式？

```css
/* 问题：子元素浮动后父元素高度塌陷 */
.container {
  /* 没有高度，因为浮动元素不占空间 */
}

/* 方案1：clearfix 伪元素（推荐） */
.clearfix::after {
  content: '';
  display: table;   /* 或 block */
  clear: both;
}
.clearfix { *zoom: 1; }  /* IE6/7 兼容 */

/* 方案2：overflow: hidden（BFC） */
.container { overflow: hidden; }
/*缺点：可能裁切超出的内容（如下拉菜单、tooltip） */

/*方案3：父元素也浮动（不推荐） */
.container { float: left; }
/* ⚠️ 缺点：影响外层布局 */

/* 方案4：额外空标签（不推荐，语义差） */
<div style="clear: both;"></div>
```

>**现代开发**：浮动布局已过时，用 Flex 或 Grid 替代。但清除浮动仍是面试必考。

## 定位

### position 的值有哪些？各自特点？

| 值 | 脱离文档流 | 参照物 | 常见用途 |
|----|-----------|--------|---------|
| `static` | ❌ | 无（正常流） | 默认值 |
| `relative` | ❌ | **自身原始位置** | 微调位置、给绝对定位当参照 |
| `absolute` | ✅ | **最近的非 static 祖先** | 弹窗、下拉菜单、图标定位 |
| `fixed` | ✅ | **浏览器视口** | 固定导航、回到顶部、对话框 |
| `sticky` | ❌→✅ | **滚动容器/视口** | 吸顶导航、表格头固定 |

```css
/* relative + absolute 经典组合 */
.parent {
  position: relative;  /* 作为子元素 absolute 的参照 */
}
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);  /* 水平垂直居中 */
}

/* sticky 吸顶效果 */
.header {
  position: sticky;
  top: 0;         /* 滚到距顶部0时吸住 */
  z-index: 100;
  background: white;
}
/* sticky 注意事项：
   1. 必须指定 top/bottom 之一
   2. 父容器不能有 overflow: hidden/auto
   3. 父容器高度必须大于 sticky 元素高度
*/
```

> **易错点**：`fixed` 在有 `transform` 的祖先元素内会参照该祖先而非视口！

```css
/* fixed 不再参照视口 */
.transform-parent {
  transform: translateZ(0);  /* 创建了新的包含块 */
}
.child {
  position: fixed;  /* 现在参照 .transform-parent 而不是视口！ */
  top: 0;
}
```

### 如何实现元素水平垂直居中？

```css
/* ====== 已知宽高 ====== */

/* 方案1：absolute + 负 margin */
.center {
  position: absolute;
  top: 50%; left: 50%;
  width: 200px; height: 100px;
  margin-top: -50px;   /* -height/2 */
  margin-left: -100px; /* -width/2 */
}

/* 方案2：absolute + margin: auto */
.center {
  position: absolute;
  top: 0; right: 0; bottom: 0; left: 0;
  width: 200px; height: 100px;
  margin: auto;
}

/* ====== 未知宽高（推荐）====== */

/* 方案3：absolute + transform（最常用） */
.center {
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%);
}

/* 方案4：flex（最简洁） */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 方案5：grid */
.parent {
  display: grid;
  place-items: center;  /* justify-items + align-items 缩写 */
}

/* 方案6：flex + margin */
.parent { display: flex; }
.center { margin: auto; }

/* 方案7：line-height（单行文本） */
.text {
  height: 100px;
  line-height: 100px;
  text-align: center;
}
```

## Flex 布局

### Flex 布局的核心概念和常用属性？

```
主轴方向（默认水平）        交叉轴方向（默认垂直）
────────────────►         ┌─────────┐
                          │         │
  ┌──────┬──────┬──────┐  │  交叉轴  │
  │item1 │item2 │item3 │  │  方向 ↓  │
  └──────┴──────┴──────┘  │         │
                          └─────────┘

容器属性：控制整体布局
├── flex-direction    主轴方向
├── flex-wrap         是否换行
├── flex-flow         上面两个的简写
├── justify-content   主轴对齐
├── align-items       交叉轴对齐
└── align-content     多轴线对齐

项目属性：控制单个项目
├── order             排列顺序
├── flex-grow         放大比例
├── flex-shrink       缩小比例
├── flex-basis        初始大小
├── flex              上面三个的简写
└── align-self        单个项目交叉轴对齐
```

### flex: 1 到底代表什么？

```css
/* flex: 1 等价于 */
.item {
  flex-grow: 1;      /* 放大：有剩余空间时等分 */
  flex-shrink: 1;    /* 缩小：空间不足时等比缩小 */
  flex-basis: 0%;    /* 初始大小：0（忽略内容宽度） */
}

/* 对比 */
flex: auto;   /* = flex-grow:1; flex-shrink:1; flex-basis:auto; */
/* basis:auto → 先按内容宽度分配，剩余空间再按 grow 等分 */

flex: 1;      /* basis:0% → 完全忽略内容宽度，直接按 grow 等分 */
```

```
示例：3个子元素，容器 600px

flex: auto + 内容宽度分别为 100/200/300
  剩余空间 = 600 - (100+200+300) = -200 → 无剩余，不放大
  实际宽度 = 100 / 200 / 300

flex: 1 + 内容宽度分别为 100/200/300
  basis=0%，不考虑内容宽度，3等分
  实际宽度 = 200 / 200 / 200
```

> **实战选择**：等分布局用 `flex: 1`，内容驱动布局用 `flex: auto`。

### justify-content 和 align-items 的值？

```css
/* justify-content：主轴对齐 */
justify-content: flex-start;     /* ← 紧凑左对齐（默认） */
justify-content: flex-end;       /*                紧凑右对齐 → */
justify-content: center;         /*      ← 居中 → */
justify-content: space-between;  /* ←  两端对齐  → */
justify-content: space-around;   /* ←  等间距环绕  → */
justify-content: space-evenly;   /* ←  完全等间距  → */

/* space-between vs space-around vs space-evenly */
/* |■    ■    ■|   between: 两端贴边，中间等分 */
/* | ■  ■  ■ |    around:  每个两侧等距（项目间距=2×边缘距） */
/* | ■  ■  ■ |    evenly:  完全等距（所有间距相同） */

/* align-items：交叉轴对齐 */
align-items: stretch;     /* 拉伸填满（默认） */
align-items: flex-start;  /* 顶部对齐 */
align-items: flex-end;    /* 底部对齐 */
align-items: center;      /* 居中对齐 */
align-items: baseline;    /* 文字基线对齐 */
```

## Grid 布局

### Grid 布局核心属性？

```css
/* 定义网格 */
.grid {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;   /* 3列：固定 + 2等分 */
  grid-template-rows: 60px 1fr 40px;       /* 3行 */
  grid-template-areas:
    "header header header"
    "sidebar main   main"
    "footer footer footer";
  gap: 16px;    /* 行列间距 */
}

/* 区域命名 */
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* 常用简写 */
.grid {
  /* 重复 */
  grid-template-columns: repeat(3, 1fr);           /* 3等分 */
  grid-template-columns: repeat(auto-fill, 200px);  /* 自动填充 */

  /* minmax 自适应 */
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  /* 每列最小200px，有剩余则等分，自动换行 */
}

/* 项目跨行跨列 */
.item {
  grid-column: 1 / 3;    /* 从第1条线到第3条线（跨2列） */
  grid-row: 1 / 4;       /* 从第1条线到第4条线（跨3行） */
  /* 或 */
  grid-column: span 2;   /* 跨2列 */
}
```

### Flex 和 Grid 怎么选？

| 场景 | 选择 | 原因 |
|------|------|------|
| 一维排列（导航、按钮组、卡片行） | **Flex** | 单方向对齐更简单 |
| 二维布局（仪表盘、表单、复杂页面） | **Grid** | 行列同时控制 |
| 不确定项目数量 | **Flex** | 自动换行灵活 |
| 固定区域布局 | **Grid** | 区域命名直观 |
| 居中对齐 | **Flex** | 三行代码搞定 |
| 瀑布流 | **Flex** | Grid 不原生支持瀑布流 |

> **总结**：一维用 Flex，二维用 Grid，简单居中用 Flex，复杂布局用 Grid。

## 响应式与媒体查询

### 响应式设计的核心方法？

```css
/* ① 媒体查询 */
/* 移动优先（推荐） */
.container { width: 100%; }

@media (min-width: 768px) {
  .container { width: 750px; margin: 0 auto; }
}
@media (min-width: 1024px) {
  .container { width: 980px; }
}
@media (min-width: 1280px) {
  .container { width: 1200px; }
}

/* ② 弹性单位 */
/* ❌ 固定宽度 */
.box { width: 300px; font-size: 16px; }

/* 相对单位 */
.box {
  width: 50%;                    /* 百分比：相对父元素 */
  font-size: clamp(14px, 2vw, 18px);  /* clamp：最小/理想/最大 */
  padding: 2vw;                  /* vw：相对视口宽度 */
  line-height: 1.5;              /* 无单位倍数：相对自身 font-size */
}

/* ③ 响应式图片 */
img {
  max-width: 100%;    /* 不超过容器宽度 */
  height: auto;       /* 保持比例 */
}

/* ④ Viewport 单位 */
/* vw: 视口宽度的 1%    vh: 视口高度的 1% */
/* vmin: min(vw,vh)     vmax: max(vw,vh)   */

/* ⑤ CSS 容器查询（2023+） */
.card-container {
  container-type: inline-size;
}
@container (min-width: 400px) {
  .card { flex-direction: row; }   /* 容器够宽时横排 */
}
@container (max-width: 399px) {
  .card { flex-direction: column; }  /* 容器窄时竖排 */
}
```

### 移动端适配方案有哪些？

```css
/* ====== 方案1：rem + 动态 html font-size ====== */
/* JS 根据 DPR 和屏幕宽度设置根字号 */
// document.documentElement.style.fontSize = 
//   (screenWidth / 375 * 16) + 'px';  // 设计稿375px

.box { width: 10rem; }  /* 375屏 = 160px, 750屏 = 320px */

/* ====== 方案2：vw 方案（推荐，纯 CSS） ====== */
/* 设计稿 375px，1px = 100/375 = 0.2667vw */
.box { width: 26.667vw; }  /* 设计稿100px → 26.667vw */

/* 配合 postcss-px-to-viewport 自动转换 */
.box { width: 100px; }  /* 开发时写px，构建时自动转vw */

/* ====== 方案3：viewport 缩放 ====== */
/* 动态设置 meta viewport 的 scale */
/* 适合需要 1:1 还原设计稿的场景 */

/* ====== 对比 ====== */
/*
  rem 方案：兼容性好，需 JS，根字号被占
  vw 方案：纯 CSS，推荐，需 postcss 插件
  缩放方案：1:1还原，但不灵活
*/
```

## CSS 动画

### transition 和 animation 的区别？

| 对比项 | `transition` | `animation` |
|--------|-------------|-------------|
| 触发方式 | 状态变化时触发（hover/click/class切换） | 自动播放或指定触发 |
| 关键帧 | 只有起止两帧 | `@keyframes` 可定义多帧 |
| 循环 | 不可循环 | `animation-iteration-count: infinite` |
| 暂停 | 不支持 | `animation-play-state: paused` |
| 方向 | 只能正向 | `animation-direction: reverse/alternate` |
| 复杂度 | 简单过渡 | 复杂动画序列 |

```css
/* transition 简单过渡 */
.button {
  background: #E8927C;
  transition: all 0.3s ease;  /* 属性 时长 缓动 */
}
.button:hover {
  background: #d17a65;
  transform: translateY(-2px);
}

/* animation 复杂动画 */
@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}
.loading {
  animation: pulse 1.5s ease-in-out infinite;
  /* 名称 时长 缓动 循环 */
}
```

### 19. 如何做性能优化的动画？

```css
/* ❌ 触发回流的动画（不要用） */
.move {
  transition: left 0.3s, top 0.3s;
  /* left/top 变化 → 回流 → 重绘 → 性能差 */
}

/* 只触发合成的动画（推荐） */
.move {
  transition: transform 0.3s;
  will-change: transform;  /* 提前告知浏览器，创建独立图层 */
  /* transform 变化 → GPU 合成 → 性能好 */
}

/* 动画性能优化清单 */
/* 1. 只动画 transform 和 opacity */
/* 2. 加 will-change 提示（但不要滥用） */
/* 3. 避免同时动画大量元素 */
/* 4. 用 requestAnimationFrame 代替 setTimeout */
/* 5. 移除不需要的 will-change */
```

```javascript
// requestAnimationFrame 示例
function animate() {
  element.style.transform = `translateX(${x}px)`;
  if (x < 300) {
    requestAnimationFrame(animate);
  }
}
requestAnimationFrame(animate);
```

### CSS 变量的用法和注意事项？

```css
/* 定义变量 */
:root {
  --primary: #E8927C;
  --font-sm: 13px;
  --radius: 12px;
  --shadow: 0 2px 8px rgba(0,0,0,0.04);
}

/* 使用变量 */
.card {
  background: var(--primary);
  border-radius: var(--radius);
  box-shadow: var(--shadow);
  font-size: var(--font-sm);
}

/* 变量可回退 */
.card {
  color: var(--text-color, #333);  /* 不存在时用 #333 */
}

/* 变量可运算（需 calc） */
.container {
  width: calc(var(--sidebar-width) + 20px);
}

/* 变量可动态修改（JS） */
// document.documentElement.style.setProperty('--primary', '#4A90D9');

/* ✅ 变量有作用域 */
.card { --padding: 12px; }
.card-large { --padding: 20px; }  /* 只在 .card-large 内生效 */

/* 注意事项 */
/* 1. 变量名区分大小写：--Color ≠ --color */
/* 2. 不能用在属性名中：var(--prop): red  */
/* 3. 不能直接做运算：width: var(--w) + 10px  */
/* 4. 需 calc：width: calc(var(--w) + 10px) */
```

> **实战意义**：CSS 变量是实现主题切换（暗黑模式）的核心手段。

## CSS 工程化

### CSS Modules vs CSS-in-JS vs Tailwind 怎么选？

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **CSS Modules** | 编译时生成唯一类名 | 零运行时、样式隔离、学习成本低 | 动态样式需 :global |
| **CSS-in-JS** (styled-components) | JS 运行时生成样式 | 动态样式强、完全组件化 | 运行时开销、SSR 复杂、调试难 |
| **Tailwind** | 原子化 CSS 类 | 开发快、一致性好、文件小 | 类名长、需记忆、HTML 膨胀 |
| **BEM** | 命名约定 | 语义清晰、无工具依赖 | 类名长、靠自觉 |

```css
/* BEM 命名规范 */
.block { }                    /* 块 */
.block__element { }           /* 块__元素 */
.block__element--modifier { } /* 块__元素--修饰符 */

/* 示例 */
.card { }
.card__title { }
.card__title--large { }
.card__action { }
.card__action--primary { }
```

> 💡 **选型建议**：React 项目 → CSS Modules 或 Tailwind；Vue 项目 → Scoped CSS；大型团队 → Tailwind + 设计系统
