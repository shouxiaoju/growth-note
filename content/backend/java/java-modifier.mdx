# Java 修饰符
java语言的修饰符主要分两类
- 访问修饰符
- 非访问修饰符
## 访问修饰符
Java中，可以使用访问控制符来保护对类、变量、方法和构造方法的访问。Java 支持 4 种不同的访问权限。
- default (即默认，什么也不写）: 在同一包内可见，不使用任何修饰符。使用对象：类、接口、变量、方法。
- private : 在同一类内可见。使用对象：变量、方法。 注意：不能修饰类（外部类）
- public : 对所有类可见。使用对象：类、接口、变量、方法
- protected : 对同一包内的类和所有子类可见。使用对象：变量、方法。 注意：不能修饰类（外部类）。

### 默认（default）
- 含义：在同一个包内的所有类可见，但在不同包中不可见。(子类不在同一个包内也是不可见的)【只认包】
- 关键字：没有关键字。如果在声明时不写任何修饰符，即为默认权限。
- 适用范围：类、接口、成员变量、方法、构造器。
- 场景：用于包内部的协作类。例如，一个包中有多个类共同完成一个功能，其中一些辅助类不需要暴露给包外的使用者。
- 示例:
```java
// 文件：com/example/util/Helper.java
package com.example.util;

class Helper { // 默认权限，只能在 com.example.util 包内被访问
    void doWork() { ... }
}
```
### private（最严格）
- 含义：仅在当前类内部可见。
- 适用范围：成员变量、方法、构造器、内部类。不能修饰顶层类（Top-level Class）。
- 场景：用于隐藏类的内部实现细节。外部类甚至无法直接读取或修改被 private 修饰的字段，必须通过公共的 getter/setter 方法访问。
- 示例：
```java
public class User {
    private String password; // 只有 User 类自己能访问

    public void setPassword(String pwd) {
        // 可以在这里做加密逻辑
        this.password = pwd; 
    }
}
```
### protected（介于默认和 public 之间）
- 含义：在同一个包内可见，或者在不同包的子类中可见。（同包可以，不同包只要是子类也可以）【认包也认亲】
- 适用范围：成员变量、方法、构造器、内部类。不能修饰顶层类。
- 场景：主要用于继承体系。父类希望某些成员被子类复用（即使子类在不同包中），但不希望被完全不相关的类访问。
- 注意：这是最容易混淆的修饰符。很多人误以为它只是“对子类可见”，其实它对同包的非子类也是可见的。
- 示例：
```java
// 包 A: com.parent
package com.parent;
public class Animal {
    protected void breathe() { ... } // 子类可继承，同包也可用
}

// 包 B: com.child
package com.child;
import com.parent.Animal;
public class Dog extends Animal {
    public void run() {
        breathe(); // 合法：Dog 是 Animal 的子类，即使在不同包
    }
}

// 包 B 中的另一个非子类
public class Cat {
    void test() {
        Animal a = new Animal();
        // a.breathe(); // 非法：Cat 不是子类，且不在 com.parent 包
    }
}
```
### public（最宽松）
- 适含义：对所有类可见，无论是否在同一个包，是否有继承关系。
- 用范围：类、接口、成员变量、方法、构造器。
- 限制：一个 .java 源文件中只能有一个 public 类，且文件名必须与该类名完全一致。
- 场景：对外提供的 API 接口、工具类、入口类等。
- 示例：
```java
public class StringUtils {
    public static boolean isEmpty(String str) {
        return str == null || str.length() == 0;
    }
}
```
### 特殊规则与注意事项
1. 顶层类（Top-level Class）的限制
    - 顶层类（即直接定义在 .java 文件中的类，不是内部类）只能使用 public 或 默认 修饰符。
    - 如果使用 private 或 protected 修饰顶层类，编译器会报错。
        - 原因：如果类是私有的，其他类根本无法实例化它，这就失去了存在的意义（除非是内部类）。
2. 内部类（Inner Class）
    - 内部类可以使用所有四种修饰符。
    - 这使得内部类可以灵活地控制其对外部类或其他类的可见性。
3. 重写（Override）规则的约束
    - 当子类重写父类的方法时，访问权限不能降低，但可以提高：
        - ❌ 错误：父类是 public，子类重写为 protected 或 default。
        - ❌ 错误：父类是 protected，子类重写为 default 或 private。
        - ✅ 正确：父类是 protected，子类重写为 public。
        - ✅ 正确：保持相同权限。
4. 构造器的访问控制
    - private 构造器常用于单例模式（Singleton Pattern）或工具类（防止实例化）。
    - protected 构造器常用于框架设计，强制用户通过子类来扩展功能，而不能直接实例化父类。
### 最佳实践建议
1. 字段私有化：绝大多数成员变量应设为 private，通过 public 的 getter/setter 暴露必要的访问通道。这允许你在设置值时进行验证或逻辑处理。
2. 最小权限原则：始终使用能满足需求的最小访问权限。
    - 如果一个方法只在类内部用，就用 private。
    - 如果只在包内用，就用默认。
    - 不要轻易把所有东西都设为 public，这会破坏封装性，导致后续重构困难。
3. 慎用 protected：除非你明确打算让这个类被继承，并且希望子类访问该成员，否则优先使用 private + public 方法组合。过度使用 protected 会让继承层次变得脆弱。
4. 接口默认为 public：接口中的方法和字段默认都是 public 的（即使你不写），这是由接口的语义决定的。
## 非访问修饰符
- static 修饰符，用来修饰类方法和类变量。
- final 修饰符，用来修饰类、方法和变量，final 修饰的类不能够被继承，修饰的方法不能被继承类重新定义，修饰的变量为常量，是不可修改的。
- abstract 修饰符，用来创建抽象类和抽象方法。
- synchronized 和 volatile 修饰符，主要用于线程的编程。
### static 修饰符
- 静态变量： static 关键字用来声明独立于对象的静态变量，无论一个类实例化多少对象，它的静态变量只有一份拷贝。 静态变量也被称为类变量。局部变量不能被声明为 static 变量。
- 静态方法： static 关键字用来声明独立于对象的静态方法。静态方法不能使用类的非静态变量。静态方法从参数列表得到数据，然后计算这些数据。
### final 修饰符
- final 变量： inal 表示"最后的、最终的"含义，变量一旦赋值后，不能被重新赋值。被 final 修饰的实例变量必须显式指定初始值。
  final 修饰符通常和 static 修饰符一起使用来创建类常量。
- final 方法： 父类中的 final 方法可以被子类继承，但是不能被子类重写。 声明 final 方法的主要目的是防止该方法的内容被修改。 如下所示，使用 final 修饰符声明方法。
- final 类： final 类不能被继承，没有类能够继承 final 类的任何特性。
### abstract （抽象）修饰符
1. 抽象类：抽象类不能用来实例化对象，声明抽象类的唯一目的是为了将来对该类进行扩充。
   一个类不能同时被 abstract 和 final 修饰。如果一个类包含抽象方法，那么该类一定要声明为抽象类，否则将出现编译错误。 抽象类可以包含抽象方法和非抽象方法。
```java
abstract class Caravan{
   private double price;
   private String model;
   private String year;
   public abstract void goFast(); //抽象方法
   public abstract void changeColor();
}
```
2. 抽象方法： 抽象方法是一种没有任何实现的方法，该方法的具体实现由子类提供。 抽象方法不能被声明成 final 和 static。 任何继承抽象类的子类必须实现父类的所有抽象方法，除非该子类也是抽象类。 如果一个类包含若干个抽象方法，那么该类必须声明为抽象类。抽象类可以不包含抽象方法。 抽象方法的声明以分号结尾，例如：public abstract sample();。
```java
public abstract class SuperClass{
    abstract void m(); //抽象方法
}
 
class SubClass extends SuperClass{
     //实现抽象方法
      void m(){
          .........
      }
}
```
### synchronized 修饰符
synchronized 关键字声明的方法同一时间只能被一个线程访问。synchronized 修饰符可以应用于四个访问修饰符。
```java
public synchronized void showDetails(){
.......
}
```
### volatile 修饰符
volatile 修饰的成员变量在每次被线程访问时，都强制从共享内存中重新读取该成员变量的值。而且，当成员变量发生变化时，会强制线程将变化值回写到共享内存。这样在任何时刻，两个不同的线程总是看到某个成员变量的同一个值。 一个 volatile 对象引用可能是 null。
```java
public class MyRunnable implements Runnable
{
    private volatile boolean active;
    public void run()
    {
        active = true;
        while (active) // 第一行
        {
            // 代码
        }
    }
    public void stop()
    {
        active = false; // 第二行
    }
}
```
通常情况下，在一个线程调用 run() 方法（在 Runnable 开启的线程），在另一个线程调用 stop() 方法。 如果 第一行 中缓冲区的 active 值被使用，那么在 第二行 的 active 值为 false 时循环不会停止。 但是以上代码中我们使用了 volatile 修饰 active，所以该循环会停止。
